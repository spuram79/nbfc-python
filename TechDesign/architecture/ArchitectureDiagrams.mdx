## 5. Loan Processing Workflow

```mermaid
sequenceDiagram
    participant C as Customer
    participant P as Web/Mobile App
    participant API as API Gateway
    participant CS as Customer Service
    participant LS as Loan Service
    participant US as Underwriting Service
    participant DS as Disbursement Service
    participant DB as Database
    
    C->>P: Apply for Loan
    P->>API: Submit Application
    API->>CS: Create Customer
    CS->>DB: Save Customer
    API->>LS: Create Loan App
    LS->>DB: Save Loan
    API->>US: Trigger Underwriting
    US->>DB: Store Assessment
    US-->>API: Risk Score
    API->>DS: Initiate Disbursement
    DS->>DB: Update Status
    DS-->>C: Funds Credited
```
